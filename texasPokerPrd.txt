德扑记账App产品需求文档（PRD）
文档版本：V1.0
发布日期：2026-04-29
产品形态：移动端H5（单页应用）
技术栈：纯前端，localStorage本地存储
视觉风格：深色主题，翠绿色品牌色

1. 项目背景与目标
扑克玩家需要一个轻量、私密、开箱即用的战绩记录工具。通过移动端H5实现“记账”体验，用户无需注册、无需后端，所有数据只保存在自己手机浏览器中。核心目标是：

帮助玩家快速记录每一次牌局的成绩、时长、盲注级别与对局标签。

通过统计卡片实时了解自己的累积盈亏、时薪、游戏时长等关键指标。

提供灵活的筛选查询与CSV导出，方便深度分析。

提供数据备份与恢复功能，避免换手机或清理浏览器导致数据丢失。

2. 用户画像与使用场景
典型用户：业余或半职业德州扑克玩家，每周打1-5次，习惯在局后立刻记账。

核心场景：

局后记账：打完一场现金局或锦标赛，打开H5快速录入成绩、时长、盲注级别等信息。

看板复盘：打开首页查看总盈亏、近期走势，判断自己是否需要调整策略或控制游戏时长。

数据导出：月底想用Excel分析自己的盈利曲线，按盲注级别、标签筛选记录，导出CSV。

换设备/备份：担心浏览器缓存被清理，定期把数据导出为备份文件，必要时再导入恢复。

4. 数据模型
4.1 单条记录结构（Session）
存储在 localStorage 的 poker_records 键下，值为JSON数组。

json
{
  "id": "唯一ID（时间戳+随机数）",
  "amount": 1234.56,        // 盈亏金额，正为盈利，负为亏损，可为0
  "duration": 3.5,          // 时长，单位小时，保留1位小数
  "blinds": "10/20",        // 盲注级别，如 "10/20"、"25/25/50" 或 "锦标赛"
  "tags": ["现金局","深筹"], // 标签数组，允许自定义标签
  "note": "牌感不错，控制情绪", // 可选备注
  "datetime": "2026-04-28T22:30:00.000Z" // ISO8601格式，按当地时间显示
}
4.2 统计卡片数据（动态计算）
首页展示的统计卡片基于所有记录实时计算：

总盈亏：所有记录 amount 之和，单位元（可配置货币符号）。

总游戏时长：duration 累加，单位小时。

平均时薪：总盈亏 ÷ 总时长（当总时长>0），保留2位小数。

总局数：记录条数（或区分session数）。

最近胜率：盈利次数 ÷ 总局数（可根据需要扩展）。

4.3 标签管理
预设标签供快速选择：现金局、锦标赛、线上、线下、深筹、短筹、常规桌、快速局、娱乐局。
支持用户新增自定义标签，输入后保存到本地标签库（存储在 poker_tags）。

5. 功能详细说明
5.1 成绩单首页
布局（由上至下）：

顶部导航栏：应用名“Poker Ledger”或“扑克记账”，右侧“添加记录”按钮（翠绿色主按钮）。

统计卡片区：2行x2列网格（移动端可横向滚动），卡片背景为略浅于底色的深色卡片，圆角，有微光边框。

总盈亏卡片：数字大字，盈利显示绿色加粗且前缀“+”，亏损显示红色前缀“-”。

总局数/游戏Session数。

总时长（小时）。

时薪（元/小时）。

最近记录列表标题：“最近记录”，右侧“查看全部”链接。

列表每条展示：日期、盲注级别、盈亏金额（颜色标识）、时长、标签缩略图（小圆角标签）。点击条目进入编辑模式（弹窗或新页面）。

交互：

点击“添加记录”按钮 → 弹窗或跳转添加记录页。

点击列表项 → 弹出详情/编辑弹窗（可修改或删除）。

下拉刷新不适用（H5无需），可提供手动刷新按钮用于重算统计数据。

5.2 添加记录
打开方式：首页浮动按钮或顶部导航栏按钮。使用底部弹出面板（Bottom Sheet）或全屏表单页。

表单字段：

盈亏金额：必填，数字输入框，键盘类型为数字带小数点，支持正负。前置“¥”符号。输入负数时可用“-”按钮。

游戏时长：必填，小时为单位，数字输入框，步进0.5小时（可手动输入任意小数）。

盲注级别：文本输入或选择预设项（如“10/20”、“25/50”、“5/10/20”、“锦标赛”），也可手动输入自定义。

标签：多选标签列表（预设+用户自定义），横向排列，选中高亮为翠绿色。支持新增标签：最后一项为“+ 新增”，输入后回车添加。

备注：可选文本域，限制200字。

日期时间：默认当前时间，可修改，使用datetime-local输入类型（或日期+时间分开选择），精确到分钟。

提交按钮：“保存记录”，翠绿色主按钮，成功保存后关闭面板并刷新首页数据。

若有编辑ID，则展示“删除记录”红色文字按钮。

验证规则：

金额必须填写，且为有效数字。

时长 > 0。

日期不能为空。

5.3 详细查询与筛选
入口：首页“最近记录”旁的“查看全部”，底部导航“记录”Tab（若有Tab）。

页面组成：

顶部筛选栏：可折叠面板，包含：

日期范围（开始日期-结束日期）。

盲注级别（多选下拉或搜索输入）。

标签（多选）。

盈亏范围（最小/最大金额输入）。

“重置”和“应用筛选”按钮。

筛选结果列表：分页或无限滚动（考虑到localStorage数据量不大，可一次性加载所有，前端筛选），每条记录展示：日期时间、盲注、盈亏额、时长、标签、备注摘要。右滑可删除或编辑。

底部操作栏：“导出CSV”按钮（翠绿色轮廓按钮）。

导出CSV逻辑：

根据当前筛选结果导出，若无筛选则导出全部。

CSV列：日期时间, 盈亏金额, 时长(小时), 盲注级别, 标签, 备注。

使用BOM确保Excel正确识别中文，生成文件名为poker_records_YYYYMMDD_HHmmss.csv。

通过创建Blob并触发下载实现。

交互细节：

点击记录条目弹出编辑弹窗（同添加记录逻辑）。

左滑删除需二次确认。

5.4 数据备份与恢复
入口：通常在设置页面或首页更多菜单中。假设底部有“设置”Tab或首页齿轮图标。

备份（导出）：

按钮：“导出备份文件”。

将 localStorage 中所有扑克数据（poker_records，poker_tags）打包为JSON对象，包含version、exportDate、records数组。

下载为 .json 文件。

恢复（导入）：

按钮：“导入备份文件”。

点击后触发文件选择器，接受 .json 文件。

解析后要求用户选择恢复模式：

合并：导入的记录追加到现有记录（基于id去重），标签库合并。

覆盖：清空现有数据，替换为导入数据。

导入前警告不可逆，要求二次确认。

导入成功后提示“成功导入X条记录”，并刷新首页。

清除数据：

在设置中提供“清除所有数据”按钮，红色警示，需输入确认文字或二次弹窗确认。

6. 界面与交互设计（深色主题+翠绿色）
6.1 视觉规范
背景色：深灰/黑底色，如 #121212 或 #1a1a1a。

卡片/列表项背景：#1e1e1e，带细微边框 #333。

品牌色（翠绿色）：#10b981（Tailwind emerald-500）或 #22c55e（green-500）。用于主按钮、高亮、选中状态、盈利数字、活跃标签。

亏损标识：红色 #ef4444，用于负金额。

中性文字：#e5e7eb 为主要文字，#9ca3af 为次要文字。

圆角：卡片 12px，按钮 8px，标签 20px（胶囊型）。

字体：系统默认中文字体，数字使用等宽或 font-feature-settings: "tnum" 对齐。

6.2 移动端适配
采用viewport width=device-width, initial-scale=1, maximum-scale=1。

所有触控目标至少 44x44px。

表格列表使用横向布局时注意信息密度，时间、盲注、盈亏可在一行展示。

弹出层（添加记录）使用底部抽屉，高度约视口80%，可下拉关闭。

6.3 动效
卡片加载轻微淡入，列表过渡。

按钮点击态：缩放 0.97，背景色加深。

避免过度动效，以轻快为主。

7. 技术实现要求
7.1 前端框架与工具
可使用纯HTML/CSS/JS，或轻量框架如Vue 3 + Vite 作为单页应用（SPA），路由使用hash模式以兼容静态部署。

状态管理简单，可直接用响应式数据驱动。

UI组件自建，或使用Tailwind CSS快速搭建深色主题。

7.2 本地存储设计
poker_records：Array，JSON序列化存储。

poker_tags：Array<string>，自定义标签库。

读写操作封装工具模块，处理JSON解析异常，初始化时若不存在则写入空数组。

数据量限制：localStorage一般5MB，预计数万条记录无压力，但仍需在导出时提供全量备份。

7.3 CSV导出
使用纯前端生成，将筛选后的记录数组转为CSV字符串。

增加BOM头 \uFEFF 确保Excel打开不乱码。

标签、备注等字段中如包含逗号或引号，需转义（用双引号包裹并内部双引号转双双引号）。

7.4 备份文件格式
json
{
  "version": "1.0",
  "exportDate": "2026-04-29T10:00:00Z",
  "records": [...],
  "tags": [...]
}
导入时校验version兼容性。

7.5 隐私与安全
所有数据仅存于用户浏览器localStorage，不发送任何网络请求。

无埋点、无用户追踪。

提醒用户定期手动备份。
